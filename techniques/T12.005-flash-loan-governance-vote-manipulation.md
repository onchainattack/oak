# OAK-T12.005 — Flash-Loan Governance Vote Manipulation (DEPRECATED)

**Maturity:** deprecated
**Superseded by:** [`OAK-T16.001`](./T16.001-vote-takeover-via-flash-loan.md)
**Deprecated:** 2026-07-17

## Why this ID is deprecated

Duplicate, and misfiled. This Technique and **OAK-T16.001 (Vote Takeover via Flash-Loan)** describe the same mechanism — same-block flash-borrow of governance tokens to carry a proposal, repaid in the same transaction. Their alias sets overlap (both list "flash-loan governance attack") and both anchor on Beanstalk, April 2022. One Technique under two IDs, splitting a single example population between them. The T16 entry is correctly parented and is retained; this one is deprecated. Its two additional worked anchors, Fortress Protocol and Elephant Money, have been carried across.

The ID is retained and not reused, per `CONTRIBUTING.md`. Anything citing `OAK-T12.005` should read [`OAK-T16.001`](./T16.001-vote-takeover-via-flash-loan.md) instead; the content moved there intact.

## What went wrong, for the record

T12 is **NFT-Specific Patterns** — its scope is attack classes structurally tied to non-fungible token semantics (per-token-id ownership, marketplace-mediated trade, royalty infrastructure, scarcity-of-edition as bait). Nothing in this Technique was NFT-specific.

The misfiling survived review because the file declared its parent under the label "Governance / Protocol Management" — a Tactic name that exists nowhere in the taxonomy. The invented name made T12 read as a governance Tactic, so nothing looked out of place.

These two entries are orphans of the T15 / T16 introduction recorded in `TAXONOMY-AUDIT.md`: T16 (Governance / Voting Manipulation) was created, the governance Techniques sitting under T12 were meant to move, and this pair did not. The migration is now complete.
