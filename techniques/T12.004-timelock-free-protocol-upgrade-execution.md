# OAK-T12.004 — Timelock-Free Protocol Upgrade Execution (DEPRECATED)

**Maturity:** deprecated
**Superseded by:** [`OAK-T16.006`](./T16.006-timelock-free-governance-execution.md)
**Deprecated:** 2026-07-17

## Why this ID is deprecated

Misfiled parent. This Technique describes a governance-layer design decision — whether a protocol's upgrade authority can execute without a mandatory timelock delay — and was parented under T12 (NFT-Specific Patterns), which does not reach it. It has moved to **OAK-T16.006 (Timelock-Free Governance Execution)** under T16 (Governance / Voting Manipulation), which is its correct parent. Content is unchanged apart from the parent, the renamed self-references, and a dangling reference fix (see below).

The ID is retained and not reused, per `CONTRIBUTING.md`. Anything citing `OAK-T12.004` should read [`OAK-T16.006`](./T16.006-timelock-free-governance-execution.md) instead; the content moved there intact.

## What went wrong, for the record

T12 is **NFT-Specific Patterns** — its scope is attack classes structurally tied to non-fungible token semantics (per-token-id ownership, marketplace-mediated trade, royalty infrastructure, scarcity-of-edition as bait). Nothing in this Technique was NFT-specific.

The misfiling survived review because the file declared its parent under the label "Governance / Protocol Management" — a Tactic name that exists nowhere in the taxonomy. The invented name made T12 read as a governance Tactic, so nothing looked out of place. This file also referenced "T12.002 (Governance Multisig Compromise)" twice — a Technique that does not exist. The real T12.002 is Fake-Mint / Counterfeit Collection. That reference was written against the same non-existent governance-flavoured T12, and is repointed in the superseding entry to **T11.003**, the nearest real home for the multisig-compromise incident class.

These two entries are orphans of the T15 / T16 introduction recorded in `TAXONOMY-AUDIT.md`: T16 (Governance / Voting Manipulation) was created, the governance Techniques sitting under T12 were meant to move, and this pair did not. The migration is now complete.
